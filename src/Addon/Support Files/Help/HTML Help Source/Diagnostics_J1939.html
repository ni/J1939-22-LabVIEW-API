J1939 Diagnostics Page

J1939 Diagnostics provides the ability to request and read diagnostic messages (DM) 1 and 2 from any device on the J1939 network. These messages are used to read the active and previously active diagnostic trouble codes (DTCs) respectively. This section also provides the ability to send DM 3 and 11. These messages are used to to clear previously active and active DTCs respectively.

Common Diagnostic Channels

Address to Query

This channel sets which J1939 device to address when sending a diagnostic message. The value -1 is used to address all devices on the J1939 bus.

Diagnostic Command

This channel sets which diagnostic message to send. Accepted values are as follows:

0 - Idle

1 - Get all J1939 Device Addresses

2 - Verify DTC

3 - Clear DTCs

4 - Clear Diagnostic Command Status

5 - Clear Diagnostic Error Code

Commands are latched and executed on value change. Once the execution of a command has started, Diagnostic Command Status will be set to 0 (false) until the execution is finished, at which point it will be set to 1 (true). Note: You must executed the Get all J1939 Device Addresses command before executing the Verify DTC or Clear DTCs commands.

Diagnostic Command Status

This channel indicates the status of the diagnostic command engine. When the channel is 0 (false), the engine is active. When the channel is 1 (true), the command engine has finished executing the latest command. This channel can then be polled after setting the Diagnostic Command to ensure the returned data is valid. The Clear Diagnostic Command Status command sets this channel to 0.

Diagnostic Error Code

This channel shows the error status from the diagnostic command engine. The Clear Diagnostic Error Code command sets this channel to 0.
